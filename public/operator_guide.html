Руководство оператора

Инструкция по работе с интернет-ресурсом VitaEquip

Данный раздел предназначен для пользователей и сотрудников, работающих с интернет-ресурсом для продажи медтехники, каталогом, заказами и административной панелью.

Пошаговая навигация

Разделы сгруппированы по основным операциям, которые выполняет пользователь.

Безопасность данных

Описаны правила работы с учетными данными, заказами и клиентской информацией.

Основы работы

Начало работы

Порядок входа и первичного ознакомления с интернет-ресурсом.

  1. Откройте главную страницу интернет-ресурса.
  2. Перейдите в каталог медицинского оборудования.
  3. Выберите нужную категорию товара.
  4. Ознакомьтесь с карточками оборудования.

Работа с заказами

Основные действия при оформлении и обработке заказа.

  1. Выберите товар из каталога.
  2. Откройте подробную информацию о товаре.
  3. Добавьте товар в заказ или корзину.
  4. Проверьте состав заказа перед сохранением.

Работа администратора

Доступ к управлению данными осуществляется через административную панель.

  1. Выполните вход под учетной записью администратора.
  2. Откройте нужный раздел панели управления.
  3. Добавьте, измените или удалите данные.
  4. Проверьте изменения в списке записей.
💡 Совет: После изменения данных рекомендуется обновить страницу или нажать кнопку обновления данных в административной панели.

Безопасность

Учетные данные

Правила безопасной работы с логином и паролем.

  1. Не передавайте логин и пароль другим пользователям.
  2. Используйте надежный пароль для входа в панель.
  3. После завершения работы нажимайте кнопку выхода.
  4. Не сохраняйте пароль на чужом компьютере.

Работа с клиентами

Клиентская информация должна обрабатываться аккуратно.

  1. Проверяйте корректность ФИО, телефона и email.
  2. Не вносите тестовые данные в рабочую базу без необходимости.
  3. Удаляйте записи только после проверки.
  4. Следите за уникальностью логина и email.

База данных

Работа с базой данных требует внимательности.

  1. Не изменяйте структуру таблиц без необходимости.
  2. Перед массовым удалением проверяйте выбранные записи.
  3. Контролируйте связи между пользователями и клиентами.
  4. Сохраняйте резервные копии важных данных.
⚠️ Важно: Удаление записей из связанных таблиц может повлиять на заказы, клиентов и состав заказов. Перед удалением необходимо убедиться, что запись не используется.

Обслуживание

Обновление данных

Поддержание актуальности информации в каталоге.

  1. Проверяйте актуальность цен на товары.
  2. Обновляйте изображения медицинского оборудования.
  3. Следите за корректностью категорий.
  4. Удаляйте устаревшие или ошибочные записи.

Контроль заказов

Периодическая проверка заказов и их статусов.

  1. Просматривайте новые заказы.
  2. Обновляйте статус заказа после обработки.
  3. Проверяйте состав заказа.
  4. Контролируйте сумму заказа.

Аналитика

Использование статистики для контроля работы интернет-ресурса.

  1. Откройте раздел статистики.
  2. Проверьте количество товаров, клиентов и заказов.
  3. Проанализируйте график продаж по месяцам.
  4. При необходимости выгрузите график в PNG.
💡 Совет: Для демонстрации работы интернет-ресурса в дипломном проекте рекомендуется заранее подготовить корректные тестовые данные.

Устранение неполадок

Данные не отображаются

Что проверить, если список товаров, клиентов или заказов пустой.

  1. Проверьте подключение к серверу.
  2. Убедитесь, что база данных запущена.
  3. Откройте API-адрес нужного раздела.
  4. Проверьте наличие записей в SQL Server.

Не отображается изображение

Порядок действий при проблемах с картинками товаров.

  1. Проверьте путь к файлу изображения.
  2. Убедитесь, что файл находится в папке uploads.
  3. Проверьте расширение файла.
  4. Загрузите изображение повторно через панель.

Клиент не создается

Частая причина — совпадение логина или email.

  1. Проверьте, нет ли такого пользователя в таблице USERS.
  2. Проверьте наличие записи в таблице CUSTOMERS.
  3. Используйте уникальный логин и email.
  4. Повторите сохранение клиента.
⚠️ Важно: Если ошибка связана с внешним ключом, необходимо проверить, существует ли запись, на которую ссылается добавляемая строка.
